## クライアント概要

- **業種：** アパレルECサイト運営
- **規模：** 従業員12名、月間受注件数 約800件
- **課題：** 問い合わせ対応に担当者1名が月40時間を費やしていた

---

## 課題の詳細

問い合わせの内訳を分析したところ、以下のパターンで全体の**約80%**を占めていました。

| 問い合わせ種別 | 割合 |
|--------------|------|
| 注文状況・配送日確認 | 38% |
| 返品・交換方法の案内 | 22% |
| サイズ・素材の質問 | 19% |
| その他定型対応 | 9% |
| 個別対応が必要なもの | 12% |

定型的な問い合わせを自動化すれば、担当者は本当に必要な案件に集中できると判断しました。

---

## 実施した施策

### フェーズ1：FAQ整備とナレッジベース構築（2週間）

既存の問い合わせ履歴500件を分析し、回答パターンをナレッジベースとして整備しました。

### フェーズ2：AIチャットボット構築（3週間）

- **使用技術：** RAG（検索拡張生成）構成、Claude API
- ECサイトの注文管理システムと連携し、注文番号で配送状況を自動取得
- 返品・交換フローはステップ形式で案内するフローを設計

### フェーズ3：テスト運用と調整（2週間）

担当者が実際に使いながら、回答精度が低いケースを洗い出し改善しました。

---

## 結果

導入から3ヶ月後の実績：

| 指標 | 導入前 | 導入後 |
|------|--------|--------|
| 月間問い合わせ件数 | 200件 | 200件（変わらず） |
| AI一次対応件数 | 0件 | 195件（97.5%） |
| 担当者対応件数 | 200件 | 5件 |
| 担当者の月間対応工数 | 約40時間 | 約2時間 |
| 顧客満足度（5段階） | 3.8 | 4.2 |

顧客満足度は**向上**しました。24時間・即時対応になったことが評価されています。
